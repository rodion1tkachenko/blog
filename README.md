# О проекте

Данный проект представляет собой полноценное веб-приложение — персональный блог с административной панелью.

Пользователи могут:

- просматривать посты автора;
- читать информацию об авторе;
- отправлять feedback через форму обратной связи.

Администратор может:

- авторизоваться через JWT;
- создавать посты;
- удалять посты;
- редактировать информацию профиля автора;
- просматривать сообщения от пользователей.

Проект разделен на frontend и backend части и реализован по клиент-серверной архитектуре.

# Основной функционал

## Публичная часть сайта

### Главная страница

На главной странице отображаются:

- информация об авторе;
- список постов;

### Feedback форма

Пользователь может отправить:

- имя;

- сообщение автору.

Сообщения сохраняются в базе данных.

---

# Авторизация

В проекте реализована JWT-аутентификация.

После успешного логина:

- сервер генерирует JWT-токен;
- токен сохраняется в localStorage;
- frontend автоматически добавляет Bearer Token в запросы администратора.

---

# Административная панель

Админ имеет возможность:

## Управление постами

- создание постов;
- удаление постов.

## Управление профилем

Редактирование:

- имени;
- университета;
- описания (bio);
- Telegram-контакта.

## Просмотр feedback

Все сообщения пользователей отображаются в админ-панели.

# Структура базы данных

В проекте используются следующие основные сущности:

## User

Хранение пользователей и ролей.

Поля:

- id
- username
- password
- role

---

## Post

Хранение постов блога.

Поля:

- id
- title
- content
- createdAt

---

## Feedback

Сообщения пользователей.

Поля:

- name

- message

---

Информация об авторе блога.

# Безопасность

Для защиты API используется Spring Security.

Реализовано:

- JWT-фильтр;
- role-based access;
- разграничение public/admin endpoint’ов;
- защита административных маршрутов.

---

# Liquibase

Для управления схемой базы данных используется Liquibase.

Liquibase отвечает за:

- создание таблиц;
- изменение структуры БД;
- version control для миграций.
